1. Introducción

1.1. Descripción General 

Fenix-SGCN es el ecosistema operativo integral diseñado para modernizar y unificar el ciclo de vida (PHVA) de un sistema de Gestión de Continuidad de Negocio para las empresas independiente de su tamaño. Superando las limitaciones de las aplicaciones GRC de gestión tradicional, Fenix-SGCN se posiciona como el socio estratégico indispensable para todas las empresas. La Aplicación está diseñada para servir a un espectro completo de usuarios: desde empresas que entregan productos y servicios, profesionales que gerencian procesos de continuidad de negocio, asesores que implementan estos sistemas de gestión hasta los auditores que preparan a las empresas para su certificación. 

1.2. Misión del Producto 
Nuestra misión es transformar radicalmente la gestión y la experiencia del ciclo de vida (PHVA) de un sistema de Gestión de Continuidad de Negocio para las Empresas, consolidando a Fenix-SGCN como la aplicación indispensable para garantizar el control de la continuidad del negocio en las Empresas. Nos dedicamos a:
* Empoderar a los Administradores: Proporcionar a los administradores del proceso de continuidad y a las empresas de gestión de continuidad y que realizan la  administración profesional, de las herramientas de automatización y análisis más potentes del mercado, permitiéndoles escalar sus operaciones, eliminar tareas manuales y ofrecer un servicio de excelencia.
* Mejorar la Calidad de del proceso de continuidad en las Empresas: Ofrecer a la alta Gerencia y al dueño del proceso de continuidad, una experiencia de usuario fluida, segura y enriquecedora que simplifique su día a día, desde la identificación de los procesos críticos, su análisis de riesgo, su análisis de impacto al negocio, su estrategia de continuidad, sus pruebas y métricas adecuadas, la confianza de conocer en línea, el estado real de la contingencia para atender una posible recuperación ante la materialización de un riesgo que genera una interrupción convirtiendo la aplicación en el centro de control de las acciones a seguir.
* Liderar con Innovación Tecnológica: Utilizar nuestra arquitectura técnica superior para ofrecer una plataforma más rápida, segura y fiable, estableciendo el nuevo estándar de lo que es posible en el sector PropTech.

2. Arquitectura y Stack Tecnológico sugerido (Usar el conjunto de versiones de dependencias mas actualizadas pero estables)

## 2.1. Arquitectura técnica recomendada
- **API-first** con OpenAPI 3.1, GraphQL opcional.  
- **Backend:** NestJS/Node.js con microservicios, Prisma ORM.  
- **Frontend:** Next.js + React + internacionalización i18next.  
- **DB:** PostgreSQL multi-tenant (schema-per-tenant), soporte DB-per-tenant para enterprise. supabase
- **Storage:** S3-compatible con versionado, retención, WORM.  
- **Orquestación:** Kafka para eventos críticos, Redis para cache.  
- **Monitoring:** Prometheus, Grafana, OpenTelemetry, Sentry.  
- **Infraestructura:** Kubernetes multi-región, Terraform, ArgoCD.  
- **Testing:** Vitest/Jest + Playwright E2E + Chaos Engineering. 
- **Autenticación: JWT (JSON Web Tokens)
- **Validación: Zod 
- **socket.io: Requerido para funcionalidades en tiempo real.
- **Gráficos y Visualización: Recharts para gráficos y estadísticas
- **Generación de PDFs: pdfkit para reportes exportables
- **Métricas de Rendimiento: Core Web Vitals (LCP < 2.5s, INP < 200ms, CLS < 0.1)

5. Resumen comprensivo del funcionamiento

Visión General del Sistema
Fénix-SGCN es un ecosistema operativo integral diseñado específicamente para modernizar y unificar la administración de Sistema de Gestión de Continuidad de Negocio. La aplicación trasciende el concepto tradicional de software de gestión para posicionarse como el socio estratégico indispensable para la Gestión de Continuidad den las empresas. Su arquitectura está diseñada para servir a un espectro completo de usuarios que incluye empresas de administración profesional de SGCN, administradores del proceso de continuidad, roles responsables de la aplicación de la continuidad, los cuales buscan una experiencia sin fricción y efectiva para la gestionar la continuidad de negocio y  que requieren una  herramienta fiable y sencilla.

La plataforma centraliza la documentación, los procesos, los planes, pruebas  y las comunicaciones, automatiza las calificaciones de Riesgo e Impacto en los procesos, optimiza la atención oportuna de un incidente de continuidad que ocasiona una interrupción en la entrega de productos y/o servicios y fomenta una cultura de resiliencia en la empresa. Su misión fundamental es transformar radicalmente la gestión y la experiencia profesional en la Gestión de continuidad,  consolidándose como la aplicación central e indispensable para cada Empresa.

Arquitectura Técnica y Stack Tecnológico

Arquitectura General
La aplicación implementa una arquitectura full-stack moderna en contenedores con separación clara entre frontend y backend, diseñada bajo el principio de multi-tenancy donde cada Empresa  opera con su propio esquema de base de datos, garantizando aislamiento de datos y escalabilidad.

Frontend
El frontend es un contenedor que está construido con Next.js 15+ y React 19+, aprovechando las características más avanzadas como Server Components y Server Actions para optimizar el rendimiento y la experiencia del usuario. La aplicación utiliza TypeScript para garantizar la robustez del código y reduce significativamente los errores en tiempo de ejecución.
El sistema de diseño está basado en Shadcn/UI, proporcionando componentes altamente personalizables y reutilizables que mantienen consistencia visual en toda la plataforma. La interfaz es completamente responsive, adaptándose a dispositivos móviles, tablets y escritorio, con soporte para modo oscuro y claro. La paleta de colores principal utiliza Índigo (#4f46e5) y blanco (#ffffff), con animaciones sutiles que mejoran la experiencia sin comprometer el rendimiento.

Backend
El backend es un contenedor que está desarrollado con Node.js y NestJS, proporcionando una arquitectura modular y escalable. Utiliza Supabase  o PostgreSQL 17+ como base de datos principal con Prisma 6+ como ORM, lo que facilita las migraciones y el manejo de esquemas complejos. La autenticación se basa en JSON Web Tokens (JWT), garantizando seguridad y escalabilidad.
Para las funcionalidades en tiempo real, la aplicación integra Socket.io, permitiendo notificaciones instantáneas y actualizaciones en vivo. Las comunicaciones externas se manejan a través de Twilio para SMS y notificaciones push.

Características Técnicas Destacadas
La aplicación implementa métricas de rendimiento optimizadas siguiendo los Core Web Vitals con objetivos específicos: LCP (Largest Contentful Paint) menor a 2.5 segundos, INP (Interaction to Next Paint) menor a 200ms, y CLS (Cumulative Layout Shift) menor a 0.1. Estas métricas aseguran una experiencia de usuario excepcional.
La seguridad es una prioridad fundamental con implementación de encriptación de datos sensibles, protección contra ataques CSRF, XSS y SQL Injection, rate limiting para prevenir ataques de fuerza bruta, validación de datos tanto en cliente como servidor, auditoría de accesos y cambios, y backups automáticos diarios.

Diseñado específicamente para empresas de administración profesional que gestionan múltiples Empresas, este portal representa una oportunidad significativa en el segmento B2B. Características de Seguridad y Privacidad
La aplicación implementa un enfoque integral de seguridad que incluye encriptación de datos sensibles de cada Empresa, protección robusta contra ataques CSRF, XSS y SQL Injection, rate limiting para prevenir ataques de fuerza bruta, validación exhaustiva de datos tanto en cliente como servidor, auditoría completa de accesos y cambios con logs detallados y backups automáticos periodicos con múltiples puntos de restauración.
En términos de privacidad, la aplicación cumple con regulaciones de protección de datos, implementa políticas claras de privacidad y términos de uso, ofrece la opción de convertir datos a datos anónimos al eliminar cuentas, permite exportación de datos personales, y requiere consentimiento explícito para uso de cookies.

Estrategia de Pruebas y Calidad
La aplicación implementa una estrategia comprehensiva de pruebas que incluye pruebas unitarias para componentes y flujos de servicios utilizando Vitest, pruebas de integración para flujos principales, pruebas End-to-End de las funcionalidades para cada portal utilizando Playwright, pruebas de rendimiento y carga, y pruebas de compatibilidad con múltiples navegadores.
El control de calidad se mantiene a través de revisión de código obligatoria, análisis estático de código, monitoreo de errores en producción, retroalimentación continua de usuarios, y métricas de calidad que incluyen cobertura de pruebas y complejidad ciclomática.

Consideraciones de Escalabilidad e Internacionalización
La arquitectura está diseñada para escalar horizontalmente con optimización de consultas a base de datos, caché estratégica para datos frecuentemente accedidos, lazy loading de componentes y datos, y particionamiento de base de datos para grandes volúmenes.
La aplicación incluye soporte inicial para español, inglés y portugués con estructura preparada para fácil expansión a otros idiomas, formatos de fecha y número según localización, y soporte para múltiples monedas incluyendo dólares americanos, real brasileño y pesos colombianos. Por defecto, la aplicación arranca en español, pesos colombianos y modo claro.

Conclusión
Fenix-SGCN representa una solución integral y sofisticada para la gestión del proceso de continuidad en las empresas, que va mucho más allá de las herramientas tradicionales de administración. Su arquitectura multi-tenant, su enfoque en la experiencia del usuario, sus características avanzadas como Gestión de flujos de pruebas y gestión documental, y su estrategia de monetización escalonada la posicionan como una plataforma líder en el sector PropTech.

Su diseño técnico robusto, su enfoque en la seguridad y privacidad, y su capacidad de escalabilidad la convierten en una solución preparada para el futuro de la gestión de Sistemas de Gestión de Continuidad de Negocio - SGCN.
